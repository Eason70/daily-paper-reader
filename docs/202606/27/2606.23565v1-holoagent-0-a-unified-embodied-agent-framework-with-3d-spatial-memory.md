## 摘要
LLM智能体在数字环境中遵循一个实用的执行循环：它们对结构化状态进行推理，调用工具，检查反馈，并修正动作。将这个循环扩展到物理机器人是困难的，因为物理执行是连续的、依赖于具身、不确定的，并受到安全约束。现有的具身人工智能系统在操作、空间理解、导航和人形控制方面取得了进步，但这些能力通常仍然是专门的模块或松散耦合的决策循环。在这项工作中，我们引入了HoloAgent-0，一个用于真实世界机器人部署的统一具身智能体框架。Embodied AgentOS将语言指令转换为可执行的技能图，调度机器人资源，监控执行，并根据运行时反馈触发澄清或重新规划。HoloAgent-0通过三个耦合层组织异构的机器人模型和控制器：用于闭环执行的Embodied AgentOS、用于物理世界基础的三维空间记忆以及用于机器人动作的具身技能。我们在真实硬件上部署HoloAgent-0，并评估其空间记忆、长时域导航以及跨运动生成、物体搜索、跨机器人协调和移动操作的闭环执行。

## Abstract
LLM agents follow a practical execution loop in digital environments: they reason over structured states, invoke tools, inspect feedback, and revise actions. Extending this loop to physical robots is difficult because physical execution is continuous, embodiment-dependent, uncertain, and constrained by safety. Existing embodied-AI systems have advanced manipulation, spatial understanding, navigation, and humanoid control, but these capabilities often remain specialized modules or loosely coupled decision loops. In this work, we introduce HoloAgent-0, a unified embodied agent framework for real-world robot deployment. Embodied AgentOS converts language instructions into executable skill graphs, schedules robot resources, monitors execution, and triggers clarification or re-planning from runtime feedback. HoloAgent-0 organizes heterogeneous robot models and controllers through three coupled layers: Embodied AgentOS for closed-loop execution, 3D spatial memory for physical world grounding, and embodied skills for robot action. We deploy HoloAgent-0 on real hardware and evaluate its spatial memory, long-horizon navigation, and closed-loop execution across motion generation, object search, cross-robot coordination, and mobile manipulation.

- **整体含义**：HoloAgent-0旨在构建一个**统一的、可部署于真实机器人的具身智能体框架**，通过将数字环境中的智能体循环适配到物理世界，弥合“具身鸿沟”（embodiment gap）。它提供了一个系统级的组织方案，使异构的机器人能力（导航、操作、运动、交互等）能够通过共享的内存、类型化的技能接口和闭环监控协同工作。

## 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程

### 核心思想
- 将长时域具身任务执行建模为一个**系统组织问题**，而非单纯的低层策略学习问题。通过三个耦合层（AgentOS、Memory、Skill）构建**可观测、可组合、可验证**的物理执行抽象，使物理技能像软件API一样可被调度、监控和重规划。

### 关键技术细节

#### （1）Embodied AgentOS（具身智能体操作系统）
- **闭环优先**：采用“观察→检索→行动→监控→更新”循环，而非一次性文本规划。
- **记忆中心**：规划前从持久化空间记忆和时序记忆中检索上下文（房间、视图、对象、目标状态等）。
- **类型化技能接口**：每个技能定义命令参数、前置条件、预期效果，并发布状态事件（进度、成功/失败模式、置信度、可恢复性等）。
- **默认可观测**：记录所有命令/状态事件、检索到的记忆、状态转移和技能结果，支持调试和重规划决策。
- 运行时层通过ROS2命令/状态总线连接AgentOS、内存层、技能层和监控验证层。

#### （2）Memory Layer（内存层）
- **空间记忆**：包含度量几何（点云/网格）、机器人位姿与定位、开放词汇3D语义地图、分层多模态场景图（HMSG）。
  - 几何记忆支持LiDAR和纯视觉两种后端（如GeoFlow-SLAM++）。
  - 语义记忆：使用SAM2生成2D掩码，计算三个SigLIP描述符（全局、掩码段、边界框），并通过加权平均融合（公式1：\( d = \sum_{i=0}^2 w_i \odot d_i \)，\(\odot\)为Hadamard积）。
  - 实例关联：将新2D观测与已有3D实例通过IoU匹配后融合或新建。
  - HMSG：分楼层、房间、视图、对象四层，支持粗到细的空间检索。
- **时序记忆**：记录目标与计划状态、执行与恢复轨迹、结果摘要，用于任务中断恢复和重规划。

#### （3）Skill Layer（技能层）
- 将机器人能力暴露为**类型化、可监控的技能调用**，如`move_to(room=kitchen)`、`pick(object=mug)`等。
- 后端包括：
  - **HoloNavi**（导航）：层级对象导航（LLM解析→HMSG匹配→VLM验证→探索），包含快速匹配和慢速推理两阶段。
  - **HoloBrain**（操作）：VLA策略，支持pick、place、fold等，返回失败模式。
  - **HoloMotion**（全身运动）：参考跟踪和速度跟踪模式，支持人形行走、跳舞等。
- 跨实体协调：通过共享内存和状态事件，AgentOS可将任务分配给不同机器人，并协调并发执行。

### 算法流程（文字说明）
1. 用户指令输入→AgentOS解析为技能图（节点为技能，边为顺序/前置条件）。
2. AgentOS从记忆层检索空间和时序上下文。
3. 调度器按依赖顺序发送技能调用（通过ROS2命令主题）。
4. 技能后端执行并持续发布状态事件（进度、成功/失败、置信度等）。
5. 监控验证层分析状态事件，决定继续、重试、重规划或请求澄清。
6. 执行后，记忆层更新空间记录和时序轨迹。

## 6. 论文的主要结论与发现

- **导航性能提升**：HoloAgent-Nav在HM3D-ObjNav仿真上取得SR 82.6%、SPL 42.8%，优于所有对比方法（包括最好的MSGNav SR 74.1%/SPL 33.4%）。在真实机器人上，Top-1@1.0m成功率97.7%，Top-5@1.0m 98.9%，显著优于FSR-VLN（91.95%/94.25%），且不同距离阈值下得分几乎一致，表明成功任务已停在严格阈值内，剩余失败需改进恢复策略。
- **语义映射竞争力**：HoloAgent-Memory在ScanNet上mIoU 31.58、f-Acc 61.58，为在线方法中最佳或接近最佳；在Replica上表现中等（mIoU 29.93），但f-mIoU和f-Acc仍具有竞争力。说明其在线语义表示可为下游规划提供有效查询接口。
- **闭环执行有效性**：定性演示表明AgentOS能够组合异构技能（导航、操作、运动），并通过状态反馈进行重规划，处理目标丢失、物体移动等变化。
- **系统组织价值**：类型化技能接口、持久化记忆和闭环监控是实现可靠物理执行的关键，而非单纯依赖更强的低层策略模型。

## 7. 优点

- **统一架构**：首次将数字环境智能体循环系统化地适配到物理机器人，提供明确的三层分离（AgentOS、Memory、Skill），便于扩展和维护。
- **可组合性**：类型化技能接口使得异构机器人（人形、轮式）可通过同一ROS2总线协作，无需暴露底层控制器细节。
- **持久化空间记忆**：HMSG结合粗到细检索和VLM验证，有效平衡效率与精度；在线更新机制使记忆能随环境变化而演变。
- **闭环重规划**：通过状态事件驱动，而非盲目执行，可处理技能失败、环境变化等不确定性。
- **真实硬件验证**：在多种机器人平台上实验，包括人形和轮式操作器，证明了框架的通用性。
- **开放性**：代码和网页已公开，促进社区复现和改进。

## 8. 不足与局限

- **缺乏标准化端到端基准**：整个AgentOS闭环（含操作、运动、跨模态）无统一量化评估，仅靠定性演示说服力有限。
- **模型泛化限制**：当前低层策略（如HoloBrain、HoloMotion）仍依赖专门后端，对宽泛自然语言指令的鲁棒性不足，AgentOS需为其适配不同接口和失败模式。
- **动态场景测试不足**：语义映射未在物体移动后定量评估更新质量；导航实验中的真实环境变化有限。
- **未报告计算开销**：未说明推理延迟或内存消耗，不利于实际部署权衡。
- **人形全栈集成尚浅**：人形平台仅展示了简单技能（舞蹈、行走），未实现完整的“导航→操作→恢复”闭环，平衡、安全等挑战未充分验证。
- **实验重复性和统计显著性**：未报告多次运行的标准差或置信区间，结果可靠性打折扣。
- **未来方向提及**：作者指出需引入数字孪生沙盒（EmbodiedGen）进行快速策略验证，但目前未包含在实验内。

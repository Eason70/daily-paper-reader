## 摘要
长期机器人部署需要一种紧凑且可扩展的记忆系统，该系统能够保留细粒度的视觉语义、在时空中锚定观察，并支持高效的存储与检索。本文提出RAVEN，一种用于长时程机器人问答与导航的智能体记忆系统。RAVEN将视觉嵌入与位姿及时间信息存储于向量数据库中，并在空间地图中实现检索，以回答查询并导航至目标。通过直接操作视觉嵌入，RAVEN避免了有损的图像到文本描述转换，并实现了大规模下准确的语义、空间与时间检索。在多个模拟及真实世界的视频问答基准测试中，RAVEN持续超越基于文本描述的记忆系统，并在长时程任务中与前沿视觉语言模型以十分之一的检索成本匹敌。最后，我们将RAVEN部署于Unitree Go1机器人上，完成自然语言目标达成的长时程导航任务，并在多个大型室内环境中展示了成功部署。

## Abstract
Long-term robot deployment requires a compact and scalable memory that preserves fine-grained visual semantics, grounds observations in space and time, and enables efficient storage and retrieval. In this paper, we propose RAVEN, an agentic memory system for long-horizon robotic question answering and navigation. RAVEN stores visual embeddings with pose and time in a vector database, and grounds retrieval in a spatial map to answer queries and navigate to goals. By operating directly on visual embeddings, RAVEN avoids lossy image-to-text captioning and enables accurate semantic, spatial, and temporal retrieval at scale. Across several simulated and real-world video question-answering benchmarks, RAVEN consistently surpasses caption-based memory systems and matches frontier VLMs on long-horizon tasks at 10$\times$ lower retrieval cost. Finally, we instantiate RAVEN on a Unitree Go1 robot for the task of long-horizon navigation for natural language goal-reaching, and show successful deployment over several large indoor environments.

## 1. 核心问题与整体含义（研究动机与背景）
机器人长期自主部署需要一种**紧凑、可扩展的记忆系统**，能够保留细粒度视觉语义，将观察在时空中锚定，并支持高效的存储与检索。现有的机器人记忆方法存在明显缺陷：
- **语言瓶颈**：主流方法（如ReMEmbR）先将视觉观测转化为文字描述（captioning），再存储文本嵌入。这一过程是有损的——形状、纹理、精确空间关系等难以语言化的细节会丢失，尤其对细粒度查询（如“带粉色蝴蝶结的蓝帽子”）脆弱。
- **显式语义地图**：依赖预定义标签，词汇有限，无法处理开放词汇查询。
- **原始视频流**：信息完整但体积过大，冗余高，无法直接作为记忆。
- **RNN/检索免方法**：存在长期遗忘或难以扩展到极长序列的问题。

**本文的核心主张**：直接利用视觉嵌入作为语义记忆的基本单元，可以绕过语言瓶颈，保留高保真视觉信息，同时借助向量数据库实现高效检索。为此，作者提出 **RAVEN（Retrieval via Visual Embeddings for Navigation）**，一个基于视觉-空间-时间三元组的智能体记忆系统，用于长时程机器人问答与导航。

## 2. 论文提出的方法论

### 2.1 核心思想
RAVEN 将机器人探索过程中采集的 RGB 视频帧，使用**多模态编码器**（如 CLIP、SigLIP、Seed1.6-Embedding、QQMM-v2）提取视觉嵌入，并结合**位姿（3D坐标）**和**时间戳**构成三元组，存入向量数据库（FAISS/Milvus）。查询时，一个 **VLM 智能体**（如 Gemini、GPT）通过迭代调用检索工具（文本、时间、位置、图像）从数据库中获取相关记忆，并基于视觉上下文推理答案，最终输出文本答案或导航目标坐标。

### 2.2 关键技术细节
- **记忆构建阶段**：编码每一帧 RGB 图像 \(o_i\) 得到嵌入 \(z_i\)，与对应位姿 \(p_i=(x_i,y_i,z_i)\) 和时间戳 \(t_i\) 组成 \(\{m_i\}=\{(p_1,t_1,z_1,o_1), ...\}\)。
- **查询阶段**：系统作为有限状态机运行；VLM agent 维护工作记忆上下文，根据当前证据充分性决定是否执行新的检索操作。提供四种工具：
  - **文本检索**：将用户查询文本编码后，与存储的视觉嵌入计算余弦相似度，返回 top-K。
  - **时间检索**：返回指定时间戳附近的连续帧。
  - **位置检索**：返回指定位置的空间最近邻。
  - **图像检索**：支持以图像作为查询（附录 B）。
- **推理循环**：每次检索返回的 K 个记忆项（包含图像、位姿、相似度分数）加入上下文，VLM 判断是否足够；若不足则继续检索，直至生成最终答案或目标坐标。
- **规划与导航**：在真实机器人上，先通过 SLAM 构建 2D 占据栅格地图，利用 A* 路径规划导航至预测位置。

### 2.3 整体流程（文字说明）
- Phase I：机器人主动探索环境（如前沿探索），同时录制 RGB-D 视频、位姿、时间。所有帧被编码为视觉嵌入，与位姿、时间存入向量数据库。
- Phase II：给定自然语言查询，VLM agent 利用检索工具从数据库中获取相关帧图像，经多轮推理后输出答案或目标位置，然后由轨迹规划器导航至该位置。

## 3. 实验设计

### 3.1 数据集 / 场景
- **NaVQA**：基于 CODa 机器人数据集构建的视频问答基准，包含短/中/长三种时间范围的描述性、空间和时间查询。
- **FindingDory**：基于 Habitat 模拟的导航基准，要求机器人从操作历史中检索记忆以完成单目标或多目标导航。
- **RAVEN-QA（自建）**：共 273 个问答对，来源包括：
  - 19 个 Habitat 模拟场景（机器人视角）
  - 4 个真实机器人场景（Unitree Go1 和 Earthrover Mini）
  - 网络爬取的室内外漫游视频（人类视角）
  - 自录的人类视角漫游视频
  - 问题类型分类：主要物体、次要物体、信息回忆、推理、空间理解。

### 3.2 基线方法
- **ReMEmbR**：使用 GPT-4o-mini 生成图像描述，存储文本嵌入进行检索。
- **VLM Only**：直接将所有帧（或子采样帧）输入 VLM，无显式检索和推理循环。
- **Embedder Only**：仅用多模态嵌入模型计算相似度，返回最匹配帧，不使用语言模型。

### 3.3 评估指标
- 准确率（descriptive、overall）、位置误差、时间误差、高层成功率、记忆效率 \(\alpha = N_{\text{retrieved}} / N_{\text{memory}}\)。

### 3.4 主要对比配置
- 多模态编码器：QQMM-v2、Seed1.6-Embed（闭源）、SigLIP、CLIP-B 等。
- VLM 智能体：Gemini-2.5-Flash、GPT-5.2、Gemini-3-Pro、Gemma3-27b、Qwen3-VL-32b 等。

## 6. 论文的主要结论与发现
- **RAVEN 持续超越基于描述的基线**：在硬查询（次要物体、信息回忆）上，RAVEN 比 ReMEmbR 提高约 30% 绝对准确率，而 ReMEmbR 常因“描述不足”或“描述过度”失败。
- **比 VLM Only 更高效且准确**：在长时程场景（>3000帧），RAVEN 比直接全帧输入 VLM 准确率更高（4%），且检索效率提高约 10 倍（α 百分比 vs 100%）。
- **紧凑可扩展**：RAVEN 在 250× 压缩下仍保持 90% 性能，相当于从原始 RGB 数据获得 22,315× 存储压缩率。
- **中等规模开源 VLM 性能不足**：当使用 27B~32B 开源 VLM 时，RAVEN 和 ReMEmbR 均不如仅用嵌入的 Eembedder Only 基线，表明小模型推理能力有限。
- **真实部署成功**：在 Unitree Go1 机器人上，RAVEN 完成导航任务，成功率高达 97.1%（SigLIP + GPT-5.2）。

## 7. 优点（方法/实验设计亮点）
- **避免语言瓶颈**：直接使用视觉嵌入，保留细粒度语义信息，无需有损的中间文字描述。
- **训练无关（training-free）**：仅利用预训练编码器和 VLM，不要求在本数据集上训练任何模块，即插即用。
- **灵活的多模式检索**：支持文本、时间、位置、图像四种检索工具，且 VLM agent 可迭代推理，有效缩短注意力窗口。
- **高度紧凑**：通过高效的向量检索和帧子采样，实现大规模压缩而不损失关键信息。
- **全面的实验覆盖**：融合模拟、真实机器人、网络视频等多种数据源，对比多种嵌入器和 VLM，并提供了详细的消融和失败案例分析。
- **开源代码和网站**：项目网站和代码已公开，可复现。

## 8. 不足与局限
- **对嵌入模型质量依赖**：检索能力受限于预训练多模态编码器的视觉-语言对齐能力。若编码器未能捕捉某些细微特征，RAVEN 仍会失败。
- **静态记忆**：当前记忆为一次性构建的数据库，不支持动态更新或剪枝。环境变化（物体移动、状态改变）会导致记忆过时。未来需要引入记忆更新机制。
- **中等规模开源 VLM 表现不佳**：尽管 RAVEN 旨在降低对 VLM 规模的依赖，但在开源 27B 级模型上最终性能仍远逊于闭源前沿模型（如 Gemini-3-Pro）。
- **导航精度限制**：文中将位置误差容忍度设为 15m（用于计算位置准确率），在小型室内场景可能不够精细。
- **实验未覆盖高度动态场景**（如人群交互、物体频繁移动），且未涉及大规模长期部署的退化和遗忘问题。
- **缺乏与基于 3D 场景表示的对比**（如 CLIP-Fields、3D-LLM-Mem），仅比较了 2D 视角的基线。

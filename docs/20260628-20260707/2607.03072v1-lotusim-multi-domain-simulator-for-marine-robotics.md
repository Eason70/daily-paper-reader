## 摘要
仿真对于海洋机器人至关重要，它在现实测试成本高昂或危险的环境中支持操作员培训、任务演练以及人机交互。现有仿真器主要侧重于自主系统，往往缺乏人在环交互和逼真的环境物理。本文介绍了LOTUSim，一个开源的实时海洋仿真器，支持空中、水面和水下机器人系统的多用户协同操作，用于协调的海军式行动。本文的第一贡献是为用户实现实时交互性能，同时确保在共享交互仿真环境中扩展到大型舰队时的可扩展性。验证展示了稳健的人在环性能，在扩展到大型异构海洋无人机群时，保持了严格的实时执行和高视觉保真度。第二贡献是一个计算高效的、基于Ekman灵感的分层水下洋流模型，该模型捕捉风驱动力、随深度变化的流动动力学，具有足够物理保真度以支持大规模仿真。与海洋再分析数据的验证表明，与常用的随机高斯-马尔科夫洋流模型相比，该模型显著提高了准确性。这些结果证实了LOTUSim作为操作员在环海洋机器人研究仿真平台的适用性。

## Abstract
Simulation is essential for maritime robotics, supporting operator training, mission rehearsal, and human-vehicle interaction in environments where real-world testing is costly or hazardous. Existing simulators focus primarily on autonomy systems and often lack human-in-the-loop interaction and realistic environmental physics. This paper introduces LOTUSim, an open-source, real-time maritime simulator supporting multi-user interaction across aerial, surface, and underwater robotic systems for coordinated naval-style operations. The first contribution of this work is enabling real-time interactive performance for users while ensuring scalability to large fleets operating within a shared interactive simulation environment. Validation demonstrates robust human-in-the-loop performance, maintaining strict real-time execution and high visual fidelity while scaling to large heterogeneous maritime drone swarms. The second contribution is a computationally efficient, Ekman-inspired layered, underwater current model that captures wind-driven, depth-dependent flow dynamics with sufficient physical fidelity for large-scale simulations. Validation against ocean reanalysis data demonstrates substantially improved accuracy compared to commonly used stochastic Gauss-Markov current models. These results confirm LOTUSim's suitability as a simulation platform for operatorin-the-loop maritime robotics research.

- **目标**：设计一个开源、实时、跨域（空中/水面/水下）的仿真平台，支持人在环交互、大规模异构舰队协同，并引入更逼真的水下洋流模型。

## 2. 方法论：核心思想与技术细节
### 2.1 系统架构
- **组成**：基于 ROS2 + Gazebo（物理引擎 + 资产调度） + Unity（渲染与交互）的分布式架构。
- **通信**：ROS2、WebSocket、TCP/IP 支持多机分布式运行；采用 Photon Unity Networking (PUN2) 实现多用户状态同步。
- **更新循环**：每个时间步，Gazebo 通过插件查询外部客户端（物理计算、渲染、交互），同步所有返回结果后推进仿真，支持用户自定义计算节点。
- **多智能体支持**：MAS 插件动态管理 agent 的生成/删除，更新顺序随机化以实现确定性；可通过 ROS2 action server 批量或单个操作。

### 2.2 水下洋流模型：Ekman 分层模型
- **动机**：取代计算低效的 CFD 和物理不真实的高斯-马尔科夫方法，同时保持实时性。
- **核心假设**：稳态流动，水柱分为三层：
  - **表层**（顶）：风应力 + 科里奥利力 → Ekman 螺旋。公式（2）给出 \(u, v\)（包括波动轨道速度与 Ekman 螺旋叠加），\(w=0\)。
  - **中层**：地转平衡，无风/底摩擦影响，公式（3）：\(u = \bar{u}, v = \bar{v}, w=0\)。
  - **底层**（底）：底部摩擦 + 科里奥利力 → 底部 Ekman 螺旋。公式（4）给出修正项。
- **参数**：
  - 科里奥利参数 \(f = 2\Omega \sin\phi\)
  - 表面风应力 \(\tau_s = C_D \rho_{air} U_{10}^2\)，其中拖曳系数 \(C_D\) 根据 Curcic & Haus (2020) 分段定义。
- **优势**：计算高效，仅需风速、纬度、深度即可快速生成三维流场；与真实海洋再分析数据对比验证准确度更高。

## 3. 实验设计
### 3.1 实时交互性能评估
- **测试平台**：Ubuntu 22.04, i9-13980HX, RTX 4090 Laptop GPU (16GB VRAM), CUDA 12.8, Gazebo Harmonic, ROS2 Humble。
- **指标**：
  - 视觉帧率 (FPS)
  - 物理更新率（控制回路延迟）
  - 实时因子 (RTF = 仿真时间/实际时间)
- **场景**：
  - 单一类型 agent 规模测试：LRAUV（可达750个）、BlueROV（450个）保持 HITL 响应性（200ms 内）；物理延时 30ms 限制下支持 30 BlueROV 或 55 LRAUV。
  - 大规模异构场景：1 DTMB + 3 突击艇 + 3 WAMV（水面） + 45 LRAUV + 45 BlueROV（水下） + 90 X500（空中），保持 RTF=1。
- **结果**：FPS > 140，RTF 在25个 BlueROV 以下>1，所有场景满足实时要求。

### 3.2 水下洋流模型对比
- **数据源**：Copernicus 法国布雷斯特海域实测数据（-6.25°~-6°E, 46.6°~47°N），深度 0.5–1000 m，每日4次采样（00:00, 06:00, 12:00, 18:00），选取5个代表性日期（2023.11.04, 2024.06.03, 2024.07.31, 2024.08.07, 2024.10.03）。
- **对比方法**：
  - 高斯-马尔科夫模型（UUVSim/DAVE 使用）
  - Ekman 分层模型（本工作）
- **指标**：
  - MAE, RMSE
  - 归一化相对误差 (Rel MAE, Rel RMSE)
  - 改善比：\(I_{MAE} = MAE_{Ekman} / MAE_{Gauss}\), \(I_{RMSE}\) 类似
- **结果**：
  - 全深度、全天次下，Ekman 模型 MAE 和 RMSE 降低 40%–85%，浅层/中层改善最显著（>70%），深层改善约60%以上。

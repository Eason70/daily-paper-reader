## 摘要
当前的视觉-语言-动作（VLA）模型在机器人操控方面表现出色，但由于依赖即时观察，在处理需要长期记忆和推理的非马尔可夫任务时往往面临困难。现有解决方案面临“频率-能力悖论”：更强的推理模型在实时控制中速度过慢，而更快的模型又缺乏足够的推理能力。为解决这一架构不匹配问题，我们提出了HiMe，一种分层具身记忆框架，将具身智能解耦为高频执行的执行器、工作记忆的哨兵以及长期策略的规划器。我们还引入了一种基于跨模态语义模式和主动管理机制的动态知识系统，使机器人能够通过“添加、更新和删除”操作保持记忆可塑性。这种分层设计有效平衡了实时执行与慢速思考规划之间的冲突，显著提升了长时域任务的成功率。实验表明，该方法不仅优于平面记忆基线，还展现出基于人类偏好自我修正内部知识的新颖能力。

## Abstract
Current Vision-Language-Action (VLA) models excel at robotic manipulation but often struggle with non-Markovian tasks requiring long-term memory and reasoning due to their reliance on immediate observations. Existing solutions face a ''frequency-competence paradox,'' where stronger reasoning models are too slow for real-time control, while faster models lack sufficient reasoning capabilities. To resolve this architectural misalignment, we propose HiMe, a Hierarchical Embodied Memory framework that decouples embodied intelligence into a high-frequency Executor for execution, a Sentry for working memory, and a Planner for long-term strategy. We also introduce a dynamic knowledge system based on cross-modal semantic schemas and active management mechanisms, allowing robots to maintain memory plasticity through ''Add, Update, and Delete'' operations. This hierarchical design effectively balances the conflict between real-time execution and slow thinking planning, significantly improving success rates in long-horizon tasks. Experiments demonstrate that this approach not only outperforms flat memory baselines but also exhibits the novel ability to self-correct its internal knowledge based on human preferences.

- **频率-能力悖论**：更强的大视觉语言模型（VLM）推理能力强但速度慢，无法满足实时控制的高频低延迟要求；而轻量级VLM速度快但推理能力不足。
- **研究动机**：模仿人类认知的多存储结构（感觉记忆→工作记忆→长时记忆），将具身智能解耦为不同时间分辨率的三个功能层，解决实时执行与慢思考规划之间的冲突。
- **整体含义**：提出一种分层具身记忆框架（HiMe），不仅提升长程任务成功率，还使机器人具备基于人类偏好自我修正内部知识的能力，推动自主自适应机器人的发展。

## 2. 方法论

### 2.1 核心思想
- **分层解耦**：将机器人控制解耦为三层，具有不同时间分辨率：
  - **Executor（执行器）**：高频VLA，控制瞬时记忆（transient memory），负责感觉-运动协调。
  - **Sentry（哨兵）**：轻量级VLM，管理工作记忆（working memory），异步过滤连续感觉流，识别关键状态转换，触发Planner。
  - **Planner（规划器）**：重量级VLM，管理情景记忆（episodic memory），在Sentry检测到的离散节点被调用，进行深度战略推理。
- **跨模态语义模式（Cross-modal Semantic Schemata）**：以对象为中心的表示，视觉经验与高密度文本描述深度关联，支持检索隐含的所有权、程序规则和人类偏好。
- **主动管理机制（Active Management）**：引入显式的Add、Update、Delete操作，使机器人能更新过时信念、清除冗余错误数据，保持记忆可塑性和一致性。

### 2.2 关键技术细节
- **问题形式化**：长时域操控任务为部分可观测的序列决策过程。智能体在T大时间步上操作，目标是根据自然语言指令l和观测序列产生动作序列。
- **分层策略分解**：
  - 基础执行器π_e：高频率反应式策略，仅依赖当前观测o_t和激活的子目标τ_t预测动作a_t。
  - 敏捷哨兵π_s：每隔n_m个时间步评估子目标完成情况，输出门控信号u_t∈{0,1}。当检测到子目标完成或失效时（u_t=1）触发规划器。
  - 推理规划器π_p：被调用时进行两步更新：
    - 步骤1：上下文检索——根据指令和近期观测编码查询q_t，从情景记忆中检索最相关的上下文条目M_ret。
    - 步骤2：内存更新与重规划——联合更新上下文记忆（通过Add/Update/Delete）和程序记忆（生成更新后的子目标列表）。
- **记忆结构**：
  - 瞬时记忆T_t：立即观测o_t。
  - 工作记忆W_t：最近h_s个观测的滑动窗口（h_s=8），用于子目标状态验证。
  - 情景记忆E_t：包括上下文记忆（多模态键值存储）和程序记忆（有序子目标列表，带有状态标签如'done'/'active'/'pending'）。

### 2.3 实现细节
- **模型选择**：Planner采用GPT-4o；Sentry采用Qwen3-VL-8B；Executor基于π0.5微调。
- **记忆后端**：向量数据库（OpenAI text-embedding-3模型编码文本查询）进行语义检索。
- **参数设置**：缓冲区大小h_s=8，监控间隔n_m=5；Executor运行频率2Hz，预测10个动作（10Hz）并开环执行5个；Sentry每10个Executor步骤查询一次。

## 3. 实验设计

### 3.1 任务场景
- **Object Search（物体搜索）**：机器人需检查不透明盒子推断存储规则、整理散落物品，动态更新记忆（用户引入新玩具），最后依据记忆的用户偏好检索正确物体。
- **Counting（计数）**：机器人需解读视觉食谱，根据用户指定的份数准备材料；随后用户引入偏好，机器人需综合新约束和食谱记忆分配个性化食材。
- **Rearrangement（重排列）**：两阶段“清理与恢复”：先收集所有玩具放入存储箱，然后恢复原状，需回忆用户预先提供的放置偏好和原始空间配置。

## 6. 主要结论与发现

1. **HiMe显著优于所有基线**：平均任务进度达到90%，接近人类上界（94%）。而Transient Memory仅14%，Transient Memory w/ Sentry提高至26%，Flat Memory为65%，HiMe w/o Sentry为68%。
2. **Sentry的核心价值**：
   - 强制执行规划器的**时间一致性**，减少因高频重评估导致的频繁子任务切换（从14%→26%提升）。
   - 通过聚合整个子任务历史的高质量工作记忆，提供全局视角，减少幻觉和冗余重规划（68%→90%提升）。
3. **跨模态记忆优于单模态**：仅文本或仅图像记忆均不如两者结合（平均90% vs. 76%/80%）。图像提供空间定位和细粒度识别，文本提供语义和逻辑推理。
4. **主动记忆管理必要性强**：FIFO简单丢弃旧信息导致性能最差（68%）；不平管理虽保留所有历史但冗余与不一致降低性能（86%）；主动Add/Update/Delete实现最佳（90%）。
5. **效率优势显著**：HiMe相比Flat Memory减少约3倍API调用（例如Object Search中从5.4降至1.8），由于Sentry将规划器调用从每步变为稀疏子任务级，且无限记忆避免了FIFO的早期遗忘。
6. **Sentry具有保守性**：高精确率（约82%）但低召回率（约35%），倾向于不触发，因此设计了固定间隔的规划器回退机制防止无限循环。

## 7. 优点

1. **创新性分层记忆架构**：借鉴人类认知的多存储器模型，将具身智能解耦为三层（执行/哨兵/规划），有效解决实时性与推理能力的矛盾。
2. **跨模态语义知识系统**：将视觉经验与文本描述深度关联，支持检索隐含的用户偏好和规则，比纯视觉或纯文本记忆更强大。
3. **主动记忆管理机制**：显式的Add/Update/Delete操作使记忆可塑，能适应动态环境和用户变更，自我修正能力是先前工作不具备的。
4. **设计优雅**：Sentry作为轻量级VLM实现了高效的门控，大幅减少对重量级Planner的调用次数，同时保证任务连贯性。
5. **实验比较全面**：涵盖了多种类型的任务和广泛的基线，消融实验系统，验证了每个组件的必要性。
6. **开源代码和模型**：提供了代码仓库，提升了可复现性。

## 8. 不足与局限

1. **实验仅限于简单操控技能**：所有任务仅涉及拾取-放置等基本操作，未验证更复杂的灵巧操控（如拧瓶盖、装配等）。
2. **缺乏标准模拟基准**：仅在真实机器人上评估，缺乏类似MetaWorld、RLBench等模拟基准上的可控对比，影响可复现性和公平比较。

9. **任务设定受限**：环境固定、物体已知，未在更开放、杂乱的真实场景验证。

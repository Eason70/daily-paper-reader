## 摘要
动态环境中的安全导航在系统动力学未知且执行器输入受限的情况下具有挑战性。现有方法要么依赖精确模型，要么需要在线优化，或者没有明确考虑输入约束。本文针对未知欧拉-拉格朗日系统提出了一种实时控制框架，在尊重执行器限制的同时保证有限时间到达-避开-停留（FT-RAS）规范。我们通过将输入约束纳入控制器设计，扩展了时空管道（STT）框架，并推导出离线可验证的可行性条件，这些条件将可用控制权限与管道设计和不确定性边界联系起来。所得框架无需近似且计算高效，适用于实时实现。通过移动机器人、四旋翼飞行器和航天器的仿真以及移动机器人的硬件实验对所提方法进行了验证，展示了在满足执行器约束的同时实现安全导航。

## Abstract
Safe navigation in dynamic environments is challenging when system dynamics are unknown and actuator inputs are limited. Existing methods either rely on accurate models, require online optimization, or do not explicitly account for input constraints. This paper presents a real-time control framework for unknown Euler-Lagrange systems that guarantees finite-time reach-avoid-stay (FT-RAS) specifications while respecting actuator limits. We extend the spatiotemporal tube (STT) framework by incorporating input constraints into the controller design and derive offline-verifiable feasibility conditions that relate the available control authority to the tube design and uncertainty bounds. The resulting framework is approximation-free and computationally efficient, making it suitable for real-time implementation. The proposed approach is validated through simulations on a mobile robot, a quadrotor, and a spacecraft, together with hardware experiments on a mobile robot, demonstrating safe navigation while satisfying actuator constraints.

### 2. 论文提出的方法论：核心思想、关键技术细节

*   **核心思想**：
    1.  **扩展STT框架**：将输入约束作为核心设计考量，重新设计STT的中心和半径动力学，确保生成的参考轨迹（管道）与可用控制权限兼容。
    2.  **离线可验证条件**：推导出联系执行器限制、系统不确定性界和管道设计参数的一组**可行性条件**。只要这些条件在设计时得到满足，就能保证控制器在运行时能跟踪上管道并满足输入约束。
    3.  **无近似闭环控制器**：设计一个两级、闭式、无需在线优化的控制器，始终在输入限制内，并能保证系统状态严格位于STT管道内。

*   **关键技术细节**：
    1.  **STT设计**：
        *   **管道中心动力学**：
            *   **无障碍时**：中心以固定速率 `k1` 朝向目标点移动。
            *   **有障碍物接近时**：在目标驱动力基础上，叠加一个与障碍物方向`m(j)`和切线方向`v(j)`相关的避障项（权重由与障碍物的距离`d(j)`决定），确保中心始终与障碍物保持最小安全距离`r_min`。
        *   **管道半径动力学**：半径`r(t)`自适应变化。当管道靠近障碍物时，半径`r(t)`平滑收缩至一个更小的值（以保证不与障碍物碰撞）；远离时则扩展回最大值`r_max`。该半径始终正定。
    2.  **控制律设计（两级）**：
        *   **第一阶段（虚拟速度控制）**：设计参考速度`vr(t)`，其幅度被`¯v`限制。它使用一个**有界变换函数**`Ψ`，使得当系统状态`x(t)`接近管道边界（归一化误差`e1 → 1`）时，速度指令会迅速增大，将状态拉回管道内，但**始终不超过预设的最大值`¯v`**。
        *   **第二阶段（加速度/力控制）**：设计实际控制力矩`τ(t)`，其幅度被最大扭矩`¯τ`限制。它使用同样的有界变换函数`Ψ`，作用于速度跟踪误差`ev`，确保`ev`始终被限制在一个指数衰减的漏斗形约束`ρv(t)`内。这使得系统状态能快速、无超调地跟踪参考速度`vr`。
    3.  **可行性条件（核心贡献）**：
        *   条件一（速度层）： `¯v ≥ ∥pv∥ + ¯c + ¯r`。 其中`¯v`是最大参考速度，`∥pv∥`是初始速度跟踪误差界，`¯c`和`¯r`分别是管道中心和半径变化率的上界。该条件保证了管道变化的速率不会快于控制器所能提供的最大速度。
        *   条件二（加速度/力矩层）： `¯τ ≥ 1/m (V_max^M + m_i ¯d + μ_v(p_v - q_v) + a_r e_n)`。 其中`¯τ`是最大力矩，`m`是惯量相关参数，`V_max^M`是未知动力学项的上界，`m_i ¯d`是扰动项，其余是控制器和管道动态相关的参数。该条件保证了控制器有足够的力矩去克服系统惯性、未知项和动态参考速度的变化。

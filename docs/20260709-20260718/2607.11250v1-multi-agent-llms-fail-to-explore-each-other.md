## 摘要
探索对于多智能体系统的可靠自主性至关重要，然而目前尚不清楚大语言模型（LLM）智能体在相互交互时能否有效探索。我们发现，现代LLM智能体往往无法做到这一点，它们常常表现出短视和两极分化的交互模式，导致次优协调和更高的遗憾。我们将这一挑战形式化为多智能体探索问题，将其建模为部分可观测随机博弈（POSG）问题，其中智能体必须探测同伴以推断其能力并识别有效的交互策略。为解决此问题，我们引入了多智能体上下文探索（MACE），这是一个轻量级框架，通过结构化的同伴选择明确促进探索。在上下文和参数多样性设置中，MACE显著改善了探索行为和下游任务性能。我们进一步从理论上证明，探索的价值随智能体多样性的增加而增加。总体而言，我们的结果揭示了当前LLM智能体的一个基本局限性，并强调了明确引导的探索对于可靠的多智能体自主性的重要性。代码将在https://github.com/deeplearning-wisc/mace发布。

## Abstract
Exploration is essential for reliable autonomy in multi-agent systems, yet it remains unclear whether large language model (LLM) agents can explore effectively when interacting with one another. We show that modern LLM agents fail to do so, often exhibiting myopic and polarized interaction patterns that lead to suboptimal coordination and increased regret. We formalize this challenge as the Multi-Agent Exploration problem, modeling it as a partially observable stochastic game (POSG) problem in which agents must probe peers to infer their capabilities and identify effective interaction strategies. To address this, we introduce Multi- Agent Contextual Exploration (MACE), a lightweight framework that explicitly promotes exploration through structured peer selection. Across both contextual and parametric diversity settings, MACE substantially improves exploration behavior and downstream task performance. We further show theoretically that the value of exploration increases with agent diversity. Overall, our results highlight a fundamental limitation of current LLM agents and underscore the importance of explicitly guided exploration for reliable multi-agent autonomy. Code will be released in https://github.com/deeplearning-wisc/mace

# 论文总结：多智能体大语言模型未能相互探索

## 1. 核心问题与整体含义（研究动机和背景）

- **研究动机**：探索是自主智能体系统的基础能力，特别是在多智能体环境中，智能体需要通过交互来了解同伴的能力并发现有效的协作策略。然而，当前的大语言模型（LLM）智能体在多智能体交互中是否能够有效探索尚不明确。
- **核心问题**：作者通过一个简单的两臂老虎机实验发现，LLM 智能体（包括 Qwen2.5-7B、GPT-4、GPT-5）在需要平衡探索与利用时，呈现出严重的过早承诺（premature commitment）和两极分化行为，往往在几轮内锁定一个同伴且几乎不改变，即使该同伴并非最优。这一缺陷是结构性的，而非能力不足。
- **整体含义**：多智能体系统的可靠自主性依赖于智能体的探索能力，但当前 LLM 无法自主实现这一点，必须通过显式算法引导才能实现有效探索。

## 2. 方法论：核心思想、关键技术细节

- **核心思想**：将多智能体探索问题形式化为部分可观测随机博弈（POSG），并提出 **多智能体上下文探索（MACE）** 框架。MACE 将联合探索问题分解为每个智能体独立的上下文赌博机问题，通过结构化的同伴选择显式促进探索。
- **关键技术细节**：
  - **形式化建模**：每个智能体选择同伴视为拉动一支臂（arm），期望奖励建模为线性函数：`μ_{i,a,t} ≈ x_{i,a,t}^T θ_{i,a}`，其中 `x_{i,a,t}` 是编码智能体 i 与同伴 a 之间交互关系的上下文特征向量。
  - **关系特征设计**：特征包括：（i）响应多样性（基于n-gram Jaccard距离）、（ii）同伴独特性（在响应图中的中心性）、（iii）历史性能（同伴经验成功率）、（iv）交互轮次（归一化时间步），以及一个偏置项。
  - **探索策略**：采用 LinUCB 选择规则：`a_{i,t} = argmax_a [ θ̂_{i,a}^T x_{i,a,t} + α √(x_{i,a,t}^T A_{i,a}^{-1} x_{i,a,t}) ]`，其中第一项是估计奖励，第二项是不确定性奖励（鼓励探索）。A 和 b 是维护的足够统计量，通过岭回归更新参数。
  - **奖励函数**：结合改进量（后一轮分数减去前一轮）与绝对分数，取平均，既奖励改进也奖励高质量结果。
- **伪代码**：算法初始化每个智能体-同伴对的 A 和 b，对每个任务样本：每轮每个智能体计算所有同伴的上下文特征和 UCB 值，选择最大值同伴，然后生成响应，计算奖励，更新所选同伴的 A 和 b。

## 3. 实验设计

- **数据集/场景**：
  - **上下文多样性（Contextual Diversity）**：使用 HotpotQA distractor 模式（多跳问答），每个问题含2个相关证据+8个干扰段落，分配给10个 Qwen2.5-7B-Instruct 智能体，每个智能体只能看到一个段落。模拟智能体拥有不同部分上下文，需要通过探索找到有证据的同伴。
  - **参数多样性（Parametric Diversity）**：使用 Math500（4-5级难度）和 GPQA（Diamond 子集），由4个不同模型（GPT-5、Qwen2.5-7B、Llama3.1-8B、Mistral-7B）组成异构池。模拟智能体能力不同。
  - **迁移实验**：将 HotpotQA 上学到的 MACE 参数迁移到 2WikiMultihopQA 上进一步测试泛化性。
- **对比方法**：
  - **In-Context Exploration**：仅通过提示语要求 LLM 自主探索，使用与 MACE 相同的信息。
  - **Random**：每轮随机选择同伴。
  - **Pre-defined**：固定通信拓扑（如选择右侧邻居），类似多数多智能体辩论架构。
- **评估指标**：HotpotQA 用 Exact Match (EM) 和 F1；Math500 和 GPQA 用准确率。还计算累积遗憾（regret）相对于最优同伴。

## 6. 主要结论与发现

- LLM 智能体在多智能体环境中存在系统性探索不足，表现为过早锁定同伴、两极化的选择分布，且最先进的模型（GPT-5）也不例外。
- 纯粹的提示式引导（In-Context Exploration）甚至不如随机选择同伴，说明仅靠提示无法克服探索失败。
- MACE 通过结构化探索显著降低了遗憾，提升了任务性能，并且学到的策略可以泛化到未见过的任务（2WikiMultihopQA）。
- 理论分析表明：MACE 达到 O(√T log T) 的累积遗憾，而贪婪非探索策略的遗憾为 Ω(δT)，其中 δ 是智能体能力多样性。探索的价值随智能体多样性增加而增加。

## 7. 优点

- **问题新颖且重要**：指出多智能体LLM系统中的一个根本性缺陷——缺乏探索，此前未被充分认识。
- **形式化严谨**：将问题建模为POSG，理论分析给出了遗憾界并解释了探索价值与多样性的关系。
- **方法轻量有效**：MACE 基于上下文线性赌博机，不依赖复杂架构或大量计算，易于集成。
- **实验设计全面**：两种异质性（上下文、参数）、迁移测试、消融、敏感性、均质智能体对比，实验有说服力。
- **跨模型泛化**：即使在更强的模型池中（GPT-5/GPT-4 等）MACE 也表现出优势，表明方法不是依赖弱模型。
- **开源**：代码将发布，便于复现和研究。

## 8. 不足与局限

- **规模限制**：实验仅在中小规模多智能体系统中（10个或4个智能体），未评估大规模（如上百个智能体）场景，这在实际部署中可能更具挑战。
- **环境假设**：MACE 假设奖励由线性模型建模，虽然通过特征工程缓解非平稳性，但在复杂动态交互中可能存在偏差。
- **计算开销**：虽然轻量，但每个智能体需要为每个同伴维护独立的设计矩阵和参数，在智能体数量很大时存储和计算开销可能增长。
- **仅考虑合作型任务**：实验设定为任务解决协作，未涉及竞争或对抗性场景，探索需求可能不同。
- **伦理风险**：更高效的探索/协作可能被滥用，例如智能体合谋或传播误导信息，论文提及但未深入讨论防护。
- **缺乏真实多人系统测试**：实验基于标准化基准，未在真实人类-AI混合团队中验证。
